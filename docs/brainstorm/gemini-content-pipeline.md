# WildArc Content Pipeline: Gemini Image Generation → Discord → Instagram

**Date:** 2026-03-28
**Status:** Brainstorm / Architecture Design
**Owner:** Yogesh Zope

---

## 1. THE VISION

Turn WildArc's knowledge base (permaculture, bamboo, regenerative farming, Coorg ecology) into a **daily stream of beautiful, educational infographic-style images** — generated by Gemini AI, posted automatically to Discord for community feedback, and eventually to Instagram for brand building.

The two bamboo images (Structural Efficiency in Bamboo Architecture + Guide to Growing Bamboo Varieties) are the **gold standard** — rich, illustrated, educational infographics that combine technical accuracy with artistic watercolor-style presentation.

---

## 2. WHY THIS MATTERS FOR WILDARC

- **Brand Building**: Consistent visual content establishes WildArc as a thought leader in regenerative living
- **Community Engagement**: Discord becomes a living gallery where the community reacts & suggests topics
- **Content Flywheel**: Each image seeds blog posts, presentations, and social media across platforms
- **Knowledge Preservation**: Visual documentation of permaculture knowledge for the Coorg land project
- **Low Cost, High Output**: Gemini's image generation is free/cheap compared to hiring illustrators

## 3. PIPELINE ARCHITECTURE

### Layer 2: CONTENT BRAIN (What to generate?)

This is the intelligence layer that turns a raw topic into a refined Gemini prompt.

**Components:**
1. **Topic Database** — Curated list of WildArc-relevant topics organized by theme:
   - Bamboo (varieties, architecture, growing, harvesting)
   - Coffee (processing, shade-grown, value chain)
   - Permaculture (food forests, companion planting, water management)
   - Coorg Ecology (native species, Western Ghats biodiversity, monsoon patterns)
   - Regenerative Practices (composting, soil building, agroforestry)
   - Sustainable Living (natural building, off-grid systems, zero waste)

2. **Prompt Templates** — Structured prompt patterns that produce consistent style:
   ```
   STYLE: Educational infographic with watercolor illustration style
   LAYOUT: [grid / split-panel / single-scene / step-by-step]
   TOPIC: {topic}
   ELEMENTS: Technical diagrams, labeled callouts, practical tips
   PALETTE: Earthy greens, warm browns, sky blues (WildArc brand colors)
   TEXT: Include title, key facts, and actionable information
   REFERENCE: {optional reference image for style matching}
   ```

3. **Reference Image Library** — Curated folder of "style reference" images (like your bamboo architecture image) that Gemini can use to maintain visual consistency.

### Layer 3: GEMINI IMAGE GENERATION

**API Choice: Gemini 2.0 Flash (Experimental)**

Why Gemini specifically:
- Native image generation (not just DALL-E wrapper)
- Can take reference images as input (style transfer)
- Text rendering in images is decent (critical for infographics)
- Free tier is generous for this use case
- Google AI Studio API key is straightforward

**Technical Implementation:**

```typescript
// Core generation function
interface GenerationRequest {
  topic: string;
  promptTemplate: string;
  referenceImage?: Buffer;   // optional style reference
  style: 'infographic' | 'scene' | 'diagram' | 'poster';
  retryCount?: number;
}

interface GenerationResult {
  image: Buffer;
  mimeType: string;
  prompt: string;
  caption: string;
  hashtags: string[];
  metadata: {
    model: string;
    generatedAt: string;
    topic: string;
    referenceUsed: boolean;
  };
}
```

**Key API Details:**
- Endpoint: `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash-exp:generateContent`
- Auth: API key (free tier: 60 requests/minute, 1500/day)
- Input: Text prompt + optional image(s) as base64 or URI
- Output: Image as base64 in response
- Cost: Free tier should cover 5-10 images/day easily

**Prompt Engineering Strategy:**
```
You are creating an educational infographic about {topic} for WildArc,
a regenerative permaculture initiative in Coorg, India.

Style: Watercolor illustration with technical diagram overlays
Layout: {layout_type}
Must include: Title text, 3-5 key facts with labeled callouts,
practical tips section
Color palette: Forest greens (#2F6135), earth browns (#8B5E3C),
sky blues (#79BCE0), gold accents (#D8A419)
Mood: Scientific yet warm, educational yet beautiful

Topic details: {enriched_topic_description}
```

### Layer 4: POST-PROCESSING

Before distribution, each image goes through:

1. **Quality Gate** — Check image dimensions, file size, and basic quality
2. **Resize/Optimize** — Create variants:
   - Discord: 1024x1024 or 1080x1350 (portrait)
   - Instagram Feed: 1080x1350 (4:5 portrait, best engagement)
   - Instagram Story: 1080x1920 (9:16)
   - Website Gallery: 1920px wide
3. **Watermark** — Subtle WildArc logo/URL in corner
4. **Caption Generation** — Claude generates:
   - Discord caption (casual, with discussion prompts)
   - Instagram caption (hashtag-optimized, CTA included)
5. **Metadata Tagging** — Store generation params for reproducibility

**Tools:** Sharp.js (image processing), already in Node.js ecosystem

### Layer 6: DISTRIBUTION

#### Phase 1: Discord Webhook (NOW)

**How Discord Webhooks work:**
- Create a webhook URL in Discord channel settings
- POST to the URL with image + message
- No bot needed, no OAuth, no server — just a URL
- Supports embeds (rich formatting with titles, colors, fields)

```typescript
async function postToDiscord(image: Buffer, caption: string, metadata: any) {
  const formData = new FormData();
  formData.append('file', new Blob([image]), `wildarc-${metadata.topic}.png`);
  formData.append('payload_json', JSON.stringify({
    embeds: [{
      title: `🌿 ${metadata.topic}`,
      description: caption,
      color: 0x2F6135, // WildArc forest green
      footer: { text: 'Generated by WildArc Content Pipeline | Powered by Gemini' },
      timestamp: new Date().toISOString()
    }]
  }));

  await fetch(DISCORD_WEBHOOK_URL, { method: 'POST', body: formData });
}
```

**Discord Channel Structure (Suggested):**
- `#content-gallery` — Final published images
- `#content-drafts` — Images for review before posting elsewhere
- `#content-requests` — Community topic suggestions
- `#pipeline-logs` — Automated generation logs

#### Phase 2: Instagram Graph API (FUTURE)

**Prerequisites:**
- Instagram Business or Creator account
- Facebook Page linked to Instagram account
- Meta Developer App with Instagram Graph API access
- Access token with `instagram_content_publish` permission

**Flow:**
1. Upload image to a container → `POST /{ig-user-id}/media`
2. Publish the container → `POST /{ig-user-id}/media_publish`
3. Track engagement → `GET /{ig-media-id}/insights`

**Important Constraints:**
- Rate limit: 25 API calls per user per hour (publishing)
- Image must be hosted at a public URL (use Supabase Storage)
- Carousel posts: up to 10 images per post
- Caption max: 2,200 characters, 30 hashtags

#### Phase 3: WildArc Website Gallery (FUTURE)
- New route in the React app: `/gallery` or `/content`
- Pulls from Supabase storage
- Filterable by topic/theme
- Each image links to full educational content

## 6. TECHNICAL DECISIONS & TRADE-OFFS

### Why Gemini over alternatives?

| Option | Pros | Cons |
|---|---|---|
| **Gemini 2.0 Flash** ✅ | Free tier, reference image input, decent text in images, Google ecosystem | Experimental, quality inconsistent, limited control |
| DALL-E 3 | High quality, great text rendering | Paid only ($0.04-0.08/image), no reference input |
| Midjourney | Best artistic quality | No API (Discord bot only), paid, slow |
| Stable Diffusion | Free, local, full control | Setup complexity, GPU needed, poor text rendering |
| Ideogram | Best text-in-image | Paid API, smaller ecosystem |

**Verdict:** Gemini is the right choice for Phase 1. Free, API-accessible, handles reference images. Can switch to paid options if quality needs increase.

### Why Discord Webhook over Discord Bot?

| Option | Pros | Cons |
|---|---|---|
| **Webhook** ✅ | Zero infrastructure, just a URL, instant setup | One-way only (can't read reactions) |
| Bot | Two-way (commands, reactions), richer features | Needs hosting, OAuth, more complex |

**Verdict:** Start with webhook (Phase 1), add bot capabilities in Phase 3 when you need reaction tracking.

### Why not use Make.com/Zapier?

- Adds dependency on external service
- Can't customize prompt engineering deeply
- Harder to integrate with WildArc's existing Express+TypeScript stack
- The pipeline IS the product — building it teaches you the system
