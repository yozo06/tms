# 🎨 WildArc Design Specifications

This directory contains visual design specifications and wireframes for the WildArc ecosystem. 
These documents serve as the "North Star" for frontend contributors, ensuring that as new features and modules are added, the UI remains consistent, premium, and functional.

## Design Principles

1. **Information Density without Clutter**: Farmers need a lot of data quickly. Use tables, charts, and expandable drawers to keep the primary view clean.
2. **Field-First Mobile**: 80% of data entry happens in the field under bright sunlight. Use high-contrast text, large tap targets, and offline indicators.
3. **Earthy but Technical**: Use a color palette inspired by nature (forest greens, soil browns, sky blues) but maintain a crisp SaaS aesthetic. No cartoonish elements.
4. **Action-Oriented**: Every screen should make it obvious what needs attention. Use badges and color-coding for status (e.g., Red = Overdue, Yellow = Needs Action, Green = Healthy).
